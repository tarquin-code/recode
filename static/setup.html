Setup Guide

What is Plex Re-Encoder?

Plex Re-Encoder converts your video library to HEVC (H.265) format, significantly reducing file sizes while maintaining visual quality. It supports GPU-accelerated encoding via NVIDIA NVENC, Dolby Vision conversion, HDR passthrough, and integrates directly with Plex Media Server.

Dolby Vision support

Step 1: System Check

The system check detects your hardware and software. Here's what each item means:

NVIDIA GPU
Detects GPU hardware via nvidia-smi (drivers) and lspci (hardware). GPU encoding is 10-50x faster than CPU.
ffmpeg
The core encoding engine. Must be built with specific features for full functionality. Click "Build ffmpeg" if missing.
NVENC
NVIDIA's hardware encoder inside ffmpeg. Requires GPU drivers + ffmpeg built with --enable-nvenc.
libplacebo
Vulkan-based video processor. Required for Dolby Vision Profile 5 color space conversion (IPTPQc2 to BT.2020).
dovi_tool
Dolby Vision metadata tool. Extracts, converts, and injects RPU data for DV P5/P7/P8 conversion.
mkvmerge
MKV muxing tool for Dolby Vision output. Bundled with the installer — no package manager needed.
Missing items? Click the Install button next to any missing tool. ffmpeg will be compiled from source with all available features (takes 15-30 minutes). dovi_tool and mkvtoolnix are bundled as pre-built binaries — no package repos needed. mediainfo is installed via your system's package manager. Only one install can run at a time — a Cancel button allows stopping a build in progress.

Step 2: GPU Configuration

Shows detected GPUs with their VRAM and maximum concurrent encodes. The system automatically calculates capacity:

2 GB
1 encode
4 GB
2 encodes
8 GB
4 encodes
16 GB
8 encodes

Each encode uses approximately 1 GB VRAM. Multi-GPU systems distribute encodes automatically across all GPUs.

No GPU? The encoder falls back to CPU mode (libx265). CPU encoding is slower but produces slightly smaller files. No GPU is required to use the application.

Step 3: Plex Integration

If Plex Media Server is installed, the app auto-detects it and reads the authentication token from Preferences.xml. This enables:

  • Library scanning — Browse Plex libraries directly from the web UI
  • Auto library rescan — Plex automatically picks up re-encoded files
  • Webhook encoding — New media added to Plex can auto-queue for encoding
Plex Webhook URL: After setup, add this URL in Plex → Settings → Webhooks to auto-encode new media:
http://your-server-ip:9877/api/plex-webhook

Step 4: Encoding Defaults

These defaults apply when you queue files for encoding. All can be changed later in Settings.

Preset
Auto is recommended — it adjusts quality settings (CQ, bitrate, speed) based on the source resolution (4K, 1080p, SD).
Encoder
GPU (NVENC) for speed, CPU (libx265) for best compression. GPU is 10-50x faster.
Audio Codec
Passthrough copies audio untouched (required for Dolby Vision on Fire Stick). Opus gives best compression. EAC3/AC3 for AV receiver compatibility.
DV Mode
Skip leaves DV files alone. Convert to HDR10 strips DV metadata. Convert to DV P8.4 re-encodes with DV metadata for wide device support.
Temp Directory
Where encode temp files are stored. Use fast storage (SSD/NVMe) for best performance. Default: /tmp/recode
Encode Suffix
Appended to output filenames. Example: Movie_h265_HDR10_recode.mkv
Allowed Paths
Directories the app can browse. Comma-separated. Default: /mnt

Step 5: Concurrency

Concurrent
How many files to encode in parallel. Maximum is auto-calculated from GPU VRAM. Higher values use more GPU memory and power.
Auto-start
When disabled (default), the queue starts paused after a server restart. You must click Start manually. Enable for fully automated operation.

Important: Driver installation requires a server reboot. After reboot, return to the setup page to rebuild ffmpeg with GPU support (NVENC + Vulkan + libplacebo). Ubuntu/Debian uses ubuntu-drivers autoinstall for the best driver match. RHEL/Alma/Rocky automatically adds the NVIDIA CUDA repository before installing.

ffmpeg Build

ffmpeg is compiled from source to include all required features. The build automatically detects available capabilities:

With NVIDIA GPU
NVENC + Vulkan + libplacebo + libx265 + libx264 + libopus + libass
Without GPU
libx265 + libx264 + libopus + libass (CPU encoding only)

Build time: 15-30 minutes depending on your CPU. You can cancel the build at any time. Without a GPU, ffmpeg is built for CPU-only encoding (libx265). DV P5 files can still be converted to HDR10 without libplacebo — only DV P8.4 conversion requires GPU + Vulkan.
